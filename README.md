# Claude Code ✦ 飞书多维表格同步助手

> 让 Claude Code 的每一句对话，自动保存到你的飞书多维表格。

---

## 功能特性

| 功能 | 说明 |
|------|------|
| 🔄 自动入队 | Claude Code 每轮回答结束后由 Hook 自动把新消息写入本地队列 |
| ▶️ 手动启动 | 程序启动后**默认待命**，由你点「启动同步」才开始上传（不自启动服务） |
| 🧹 智能过滤 | 自动剔除斜杠命令、命令输出、caveat、系统提醒、工具调用等噪音，可按需保留 |
| 📝 导出 Markdown | 把本机所有会话导出为 `.md`（一会话一文件），纯本地不联网，账号被限也能留档 |
| 📥 历史回填 | 一键导入本机所有历史对话记录（与实时同步用同一套过滤规则） |
| 🛡 去重保障 | SQLite 本地去重，重启/断网不会重复上传 |
| ⚡ 异步队列 | 后台线程批量上传，完全不影响 Claude Code 响应速度 |
| 🔁 自动重试 | 失败最多重试 5 次，网络抖动自动恢复 |
| 🩺 一键体检 | 检测 Claude 目录/Hook/飞书连接/表格权限，人话提示问题 |
| 🧙 配置向导 | 5 步引导完成首次配置，无需手动改任何配置文件 |
| 🔒 本地存储 | 凭证存储在本机 `~/.claude_feishu_sync/`，不上传任何服务器 |
| 🔄 重置队列 | 一键清空待上传/失败任务，已同步记录与去重不受影响 |
| 📌 托盘常驻 | 关闭窗口不退出，最小化到系统托盘；托盘菜单可显示/开始/停止/退出 |

## 快速开始（3 分钟）

### 方式一：直接运行 Python 脚本（推荐先试用）

```bash
# 主程序仅用标准库即可运行，Python 3.8+
python3 feishu_sync_app.py   # macOS / Linux
python  feishu_sync_app.py   # Windows

# 可选：启用系统托盘常驻（关闭窗口最小化到托盘、托盘菜单操作）
pip install pystray pillow
```

> 未安装 `pystray pillow` 时程序仍可正常运行，只是关闭窗口会询问「退出 / 最小化」而非进入系统托盘。打包脚本已自动包含这两个依赖。

### 方式二：打包成 EXE / .app（打包一次，到处运行）

**Windows：**
```
双击运行 build_exe_win.bat
打包完成后在 dist\ 文件夹里找到「飞书同步助手.exe」
```

**macOS：**
```bash
chmod +x build_exe_mac.sh
./build_exe_mac.sh
# 打包完成后在 dist/ 文件夹里找到「飞书同步助手.app」
```

> ⚠️ **Windows SmartScreen 提示**：首次运行时可能弹出"Windows 已保护你的电脑"警告，
> 点击「更多信息」→「仍要运行」即可。这是因为程序未做代码签名，属正常现象。

## 首次配置（向导会全程引导）

程序启动后会自动弹出 **5 步配置向导**：

### 第 1 步：创建飞书自建应用

1. 打开 [飞书开放平台](https://open.feishu.cn/app)
2. 点击「创建应用」→「自建应用」，填写名称（如"Claude 同步助手"）
3. 在「权限管理」搜索并开通：**查看、评论、编辑和管理多维表格**
4. 点击「创建版本」→「申请发布」（企业管理员一键通过）
5. 在「凭证与基础信息」复制 **App ID** 和 **App Secret**

### 第 2 步：创建多维表格

向导提供两种方式：

- **自动创建（推荐）**：点击「一键创建表格」，程序自动建好表格和所有字段
- **手动填写**：使用已有多维表格，填入 App Token 和 Table ID

> ⚠️ 创建或选择表格后，**必须在飞书里完成授权**：
> 打开多维表格 → 右上角「…」→「更多」→「添加文档应用」→ 搜索并添加你的应用

### 第 3 步：注册 Hook

点击「自动注册 Hook」，程序会自动修改 `~/.claude/settings.json`。
**完成后重启 Claude Code 即可生效。**

## 启动与同步行为

- **程序启动 / 向导完成后不会自动上传**，状态显示「⏸ 待命」。需要在「状态大屏」点 **▶ 启动同步** 才开始把队列推送到飞书。
- **数据"自己冒出来"是正常的**：只要 Hook 已注册，你每结束一轮 Claude Code 对话，Hook 就会把新消息**自动写入本地队列**——这与"扫描历史"无关，也不需要你手动触发。
- 不想要这种自动入队：到「策略配置」→「彻底移除自动化 Hook」。移除后只能靠「历史回填」手动导入。

## 过滤规则（在哪把关）

过滤在**入队时**执行，实时 Hook 与历史回填走**同一套规则**；上传环节不再二次过滤。

**恒过滤（始终剔除，无需开关）：**
- 斜杠命令：`<command-name>` / `<command-message>` / `<command-args>`
- 本地命令输出：`<local-command-stdout>` / `<local-command-stderr>`
- caveat 提示、`<system-reminder>`、`<bash-input/stdout/stderr>` 等注入文本
  （若系统提醒是**追加在真实提问后面**的，只剜掉提醒、保留你的提问）

**可在「策略配置」开关：**

| 开关 | 默认 | 作用 |
|------|------|------|
| 消息角色范围 | 两者 | 仅记录 User / 仅记录 Assistant / 两者全收 |
| 过滤工具调用 | 开 | 剔除 `[工具调用: Bash]` 之类的工具触发记录 |
| 过滤工具结果 | 开 | 剔除大量终端 / 代码输出 |
| 过滤子代理消息 | 开 | 剔除 subagent 内部对话 |
| 过滤无意义短消息 | 开 | 剔除「继续 / 好的 / 嗯 / ok」等口水词 |
| 过滤助手过程旁白 | 开 | 启发式剔除「已启动…正常」「语法通过，验证一下…」等过程短句，尽量只留问题与结论 |

> ⚠️ 「过滤助手过程旁白」是基于关键词+长度的**启发式**，可能漏判或误删真正的短结论。误删了就到「策略配置」关掉这一项。
>
> ⚠️ 过滤只对**今后入队**的消息生效，**已经写入飞书的旧记录不会被回溯删除**，需自行在表格中清理。

## 导出为 Markdown

「历史回填」页 → **📝 导出为 Markdown**：把所配置路径下的所有会话导出成 `.md` 文件，
**一个会话一个文件**（命名为 `日期_项目_首句_会话短ID.md`）。

- 纯本地处理，**不联网、不经过飞书**——账号被限制 / 无法登录时也能完整留档
- 自动套用上面的去噪规则，导出内容只含你和 Claude 的正文对话

| 字段名 | 类型 | 内容 |
|--------|------|------|
| 时间 | 日期 | 消息产生的时间戳 |

## 文件结构

```
feishu_sync/
├── feishu_sync_app.py      主程序（GUI + 同步引擎 + Hook，单文件）
├── build_exe_win.bat       Windows 一键打包脚本
├── build_exe_mac.sh        macOS 一键打包脚本
└── README.md               本说明文档

运行后自动生成：
~/.claude_feishu_sync/
└── sync.db                 SQLite 数据库（WAL 模式，队列/去重/日志/配置/读取偏移）

修改的文件：
~/.claude/settings.json     Claude Code 配置（添加 Stop Hook）
```

> Hook 不再生成独立脚本：源码运行时调用 `python feishu_sync_app.py --hook`，
> 打包后直接调用 `飞书同步助手.exe --hook`，**exe 用户无需安装 Python**。

## 常见问题

**Q: 对话数据会上传到你们的服务器吗？**
A: 不会。程序直接从你的电脑调用飞书 API，数据只在你的本地和你的飞书账号之间流动。

**Q: 为什么 Hook 注册后要重启 Claude Code？**
A: Claude Code 在启动时读取配置文件，修改配置后需重启才能加载新的 Hook。

**Q: 从旧版本升级，Hook 需要重新注册吗？**
A: 需要。打开「配置」→「注册 Hook」，程序会自动把旧的 Python 脚本方式迁移为新的 `--hook` 方式，然后重启 Claude Code。

**Q: 打成 exe 后自动同步不工作？**
A: 新版 Hook 由 exe 自身处理，无需 Python。若是旧版升级，请按上一条重新注册 Hook。

**Q: 历史记录可以导入多次吗？**
A: 放心导入，程序有完整的去重机制（基于消息 UUID），不会产生重复记录。

**Q: 我还没点扫描，队列里怎么就有数据了？**
A: 那不是扫描，是 Hook 在每轮对话结束时自动入队。属正常行为；不想要可在「策略配置」移除 Hook。

**Q: 启动程序后为什么不自动开始上传？**
A: 新版默认「待命」，需手动点「▶ 启动同步」。这是为了让你掌控何时上传，避免一开就跑。

**Q: 表格里还是有 `[工具调用]` / 斜杠命令之类的噪音？**
A: 新版已恒过滤这些噪音，但只对今后入队的消息生效；旧数据需在飞书表格里自行删除。

**Q: 账号被限制 / 登录不了，聊天记录还能留下来吗？**
A: 能。用「历史回填」页的「📝 导出为 Markdown」，把所有会话导出为本地 md 文件，一会话一文件，全程不联网。

**Q: 飞书提示"应用未安装"或权限错误怎么办？**
A: 在多维表格里点「…」→「更多」→「添加文档应用」完成授权，详见配置向导第 2 步。

**Q: 飞书 API 调用频率限制是多少？**
A: 多维表格写接口为 50 次/秒，本程序批量写入并加了间隔，正常使用不会触发限流。

**Q: macOS 提示"无法打开，因为无法验证开发者"？**
A: 系统设置 → 隐私与安全 → 找到该应用 → 点「仍要打开」。

## 数据安全说明

- 所有配置保存在本地 `~/.claude_feishu_sync/sync.db`，App Secret 经机器绑定混淆后存储（非明文）
- Hook 仅从本机 JSONL 文件增量读取消息写入本地 SQLite，不做任何网络请求
- 网络请求仅在主程序运行时发起，目标仅为 `open.feishu.cn`
- 数据库为单文件本地存储，请自行保护 `~/.claude_feishu_sync/` 目录的访问权限
- 如需彻底卸载：在程序内点「移除 Hook」，再删除 `~/.claude_feishu_sync/` 目录

> 注：App Secret 的混淆密钥与本机绑定，换机器后需在「配置」中重新填写。
